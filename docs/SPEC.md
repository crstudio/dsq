# 戴森球计划 - 材料计算器

## 软件规格说明书（SRS）

### 1. 项目概述

#### 1.1 项目背景

本项目是一款基于戴森球计划游戏的材料计算器，旨在帮助玩家计算建造特定物品所需的原材料和设备。

#### 1.2 项目目标

- 提供直观的用户界面，让玩家快速选择目标物品
- 提供用户输入需要的产能数量，单位为每分钟多少个
- 支持自定义数量的计算
- 显示依赖所有的原料

#### 1.3 目标用户

- 戴森球计划游戏玩家
- 需要规划游戏内物品生产的玩家

### 2. 功能需求

#### 2.1 核心功能

##### 2.1.1 物品选择

- 用户可以通过点击按钮打开物品选择模态框
- 物品按类型分组显示（资源、材料、组件、建筑、科技等）
- 支持搜索功能，用户可以通过名称，拼音首字母或全拼或ID搜索物品
- 每个物品以图标形式展示，鼠标悬停显示物品名称

##### 2.1.2 材料计算

- 选择物品后自动计算所需原材料
- 支持自定义生产数量（默认为60）
- 计算结果保留两位小数
- 考虑配方的产出数量（ResultCounts），正确处理产出大于1的情况
- 默认使用Factories里的第1个工厂类型
- 工厂有speed属性，用于对产能进行调整，默认值为1

##### 2.1.3 结果显示

- 根据用户选择的物品和输入的产能数量，自动计算建造目标物品所需的所有原料，计算结果展示如下：
  - 结果以grid布局展示，分两列，右侧固定宽度
  - 左侧内容表格的形式展示：
    - 表格的列包括：
      - 物品
      - 产能（根据用户输入的产生需求，计算需要每分钟多少个）
      - 设备（建筑、设施和对应的数量）
      - 配方（根据配方计算所需原料及数量，每行一个配方）
      - 增产模式（默认为无）
      - 增产剂（默认为无）
      - 工厂类型（默认为Factories里的第1个，其他的选项都列出来，放在同一行）
    - 表格行的顺序按照计算迭代的顺序排序
  - 右侧显示所需要的最基本的原料有哪些，各是多少个，需要的设备树都有哪些，各是多少个

#### 2.2 用户交互

##### 2.2.1 主界面交互

- 点击"选择物品"按钮打开模态框
- 输入数字修改生产数量，默认值为60，旁边需要一个计算按钮
- 选择物品后自动计算并显示结果
- 按Escape键关闭模态框

##### 2.2.2 模态框交互

- 点击物品图标选择该物品
- 输入搜索词过滤物品列表
- 点击模态框外部或关闭按钮关闭模态框

### 3. 数据需求

#### 3.1 数据源

- 物品数据：items.json - 包含所有游戏物品的基础信息
- 配方数据：recipes.json - 包含所有物品的制造配方
- 图标资源：static/icon/Vanilla/ - 物品图标文件

#### 3.2 数据结构

##### 物品数据（Item）

```typescript
interface Item {
  ID: number;        // 物品唯一标识
  Type: number;      // 物品类型（1-资源，2-材料，3-组件，4/12-建筑等）
  Name: string;      // 物品名称
  GridIndex: number; // 网格索引
  IconName: string;  // 图标文件名
}
```

##### 配方数据（Recipe）

```typescript
interface Recipe {
  Items: number[];        // 原材料ID数组
  ItemCounts: number[];   // 原材料数量数组
  Results: number[];      // 产物ID数组
  ResultCounts: number[]; // 产物数量数组
  Name: string;          // 配方名称
}
```

### 5. 非功能需求

#### 5.1 性能需求

- 页面加载时间应小于3秒
- 计算响应时间应小于500ms

#### 5.2 兼容性

- 支持现代浏览器（Chrome、Firefox、Edge、Safari）
- 响应式设计，支持桌面端和移动端

#### 5.3 用户体验

- 界面美观，使用DaisyUI组件库
- 加载状态提示
- 错误处理和用户反馈
